题解：

 

第一题：

这题各种被秒啊囧。我围观了各种代码后发现标程的做法傻掉了，不过还是说一下吧。

首先根据期望的性质【和的期望等于期望的和】，所有结果即为所有位置的视野距离的期望之和。

令E(i,j)表示第j个人站在i这个位置，j的视野距离的期望，那么显然最终答案为sigma( sigma(E(i,j) 1 <= j <=n) / n).

下面考虑怎么计算E(i,j)，这个可以用dp解决。

f[j][a][b]表示第j个人站在了a这个位置，j前面有b个比他高的人，j的视野距离的期望。

那么显然转移的时候只考虑a-1这个位置是否站比j高的人就行了，这个可以用组合数学推出来一个关于a，b，n的公式的，详细见标程。

f数组可以在n^3的时间内预处理出来，当然也可以一边用一边算。

f数据计算之后，E(i,j) = sigma(f[j][i][k] * P_k , 0 <= k <= n)，其中P_k为j前面恰好k个比j高的人的概率，这个也是可以计算出来的。

复杂度O(n^3)

 

第二题：

令r = gcd(x,y)，x' = x / r，y' = y / r.

则原式可以化为：

x' * y' * r / (x' + y') = n!

r = n! * (x' + y') / (x' * y')

因为x'与y'互质，所以x' + y'与x'互质，x' + y'与y'互质，所以x' + y'与x' * y'互质

所以n!必能被x' * y'整除，即x'与y'必是n!的约数

所以原题的答案即为挑出两个互质的n!的约数的方法数，这个有很多做法的，我说一下标程做法

首先在nlgn的时间内对n!分解质因数（这个是很经典的算法，就不多讲了）

然后假设结果为a_1 ^ p_1 * a_2 ^ p_2 * ... * a_k ^ p_k

然后令f[i]表示从前i个质因数中挑出两个互质的约数方法数，则f[0] = 1

然后考虑第i个质因数，分三种情况：

1.都不取

2.取若干个乘进第一个数里，有p_i种可能

3.取若干个乘进第二个数里，有p_i种可能

所以f[i] = f[i - 1] + 2 * f[i - 1] * p_i

f[k]即为结果

 

第三题：

 

我们先来观察这个题的条件。

 

1、对于任意的 1<=i,j<=n, 有 A_ij = A_ji。

这就是表明了矩阵是对称的。于是我们可以将后面的条件改写成这样：

2、对于任意的 1<=i<=j<=n，有 max(L_ij, L_ji) <= A_ij <= min(R_ij, R_ji)。

3、sigma(A_ii, 1<=i<=n) + 2sigma(A_ij,1<=i<j<=n) = 0。

 

设原题中给定的系数矩阵为P。我们设 P'_ij = P_ij + P_ji (i<=j) 或 P'_ij = 0 (i > j)。

也就是说 P' 是个上三角 n 阶方阵。

同时，设 A' 是个上三角 n 阶方阵，满足 A'_ij = A_ij (i = j) 或 A'_ij = 2A_ij (i < j) 或 A'_ij = 0 (i > j)。

即 L_ij <= A'_ij <= R_ij (i = j) 或 2L_ij <= A'_ij <= 2R_ij (i < j)。

这样就只需满足 sigma(A'_ij, 1<=i<j<=n) = 0 就行了。

原来的契合度就等于 1/2 * sigma(A'_ij *P'_ij, 1<=i<=j<=n)。

由于左边是个常数，只需要最大化右边就行了。

 

然后我们发现矩阵这个条件就没用了。

相当于求一个长度为n(n+1)/2的序列 X，满足 L_i <= X_i <= R_i，且 sigma(X_i, 1<=i<=n(n+1)/2) = 0，最大化 sigma(P_i * X_i, 1<=i<=n(n+1)/2)，其中 P 是个长度为n(n+1)/2的序列。

按照 P 的大小从小到大排序，将 L 和 R 作为附加域，重排 L 序列和 R 序列。结论就是以某一个位置为界，前面的 X 都取 L 的值，后面的 X 都取 R 的值，这个位置取中间值使得总和为 0 的时候，能够取到最优值。这个用调整法很容易证明。

这样的话 O(N) 枚举中间位置即可。考虑到这样的合法位置只可能有一个，可以得到 O(logN) 的二分算法，但是木有必要……(N为矩阵大小，即n^2)

但是此题有一个特殊之处。由于矩阵 A' 的定义的限制，非对角线元素必须取偶数。

这个很容易解决。如果中间元素就是非对角线元素且要取奇数，向左找到最近的对角线元素或者向右找到最近的对角线元素，调整一下就可以了。
